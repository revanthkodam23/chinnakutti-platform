# Supabase Schema

This folder contains the content schema for Chinnakutti.Fun.

## Tables

- `languages`: supported languages, seeded with English and Telugu.
- `age_groups`: child age ranges used for filtering and recommendations.
- `categories`: bilingual category metadata, icons, and brand colors.
- `stories`: bilingual story metadata, cover image, audio narration, featured flag, SEO metadata, and publishing status.
- `story_blocks`: ordered story content blocks per language.

## Apply Locally

```bash
supabase db reset
```

or apply the SQL in `migrations/` through your Supabase project migration workflow.

## Types

`types/database.types.ts` follows the shape generated by the Supabase CLI and can be replaced later with:

```bash
supabase gen types typescript --project-id <project-id> > supabase/types/database.types.ts
```

## Admin Users

Create the user through Supabase Authentication first, then run:

`scripts/promote_auth_user_to_admin.sql`

Replace `admin@example.com` with the user's email before executing it in the
Supabase SQL Editor. The user must sign out and sign back in, or refresh their
session, before the new role appears in their JWT.
